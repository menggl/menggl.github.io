分析sql语句

使用explain查看执行计划

使用explain可以查看sql的执行计划

select_type 查询类型

  • simple 简单查询，不包含子查询或union查询
  • primary 包含复杂的子查询，在最外层的查询标记该值
  • subquery 在select或where里面包含子查询，会被标记为该值
  • derived 在from列表中包含的子查询包含这个值，Mysql会递归执行这些子查询
  • union 若第二个select出现在union之后，则会标记为改值，若union包含在from的子查询中，则外层select被标记为derived
  • union result 从union表获取结果的select

table，表示该行是关于哪张表的

partitions 匹配的分区

type 表的连接类型，其性能由高到低排序如下

  • system 表中只有一行数据的时候，相当于系统表
  • const 通过索引一次就能拿到数据，只匹配一条数据
  • eq_ref 唯一性索引扫描，对于每个索引建，都只有一条数据记录与之匹配，常用于主键或唯一索引
  • ref 非唯一性索引，返回匹配某个单独值的所有行，用于= < 或 > 操作符带索引的列
  • range 只检索给定范围的行，使用一个索引来选择行，一般使用between、> 、<情况
  • index 只遍历索引树
  • ALL 全表扫描
